# Проект: Аналитика данных для сети салонов красоты

В этом репозитории я демонстрирую навыки работы с данными, объединяя технический стек с опытом управления бизнесом.

### Что внутри:
1. **beauty_analytics1.sql**: Сложные запросы для расчета прибыли мастеров. Использую CTE и Window Functions (Rank, Partition By).
2. **data_cleaning_script.py**: Скрипт на Python (Pandas) для очистки "сырых" данных из CRM. Реализована нормализация телефонов, дат и имен.

### Мой стек:
- **SQL**: PostgreSQL / MySQL (Advanced SELECTs)
- **Python**: Pandas, NumPy, Regular Expressions
- **Инструменты**: Git, GitHub, VS Code

💇‍♂️ Beauty Analytics: Оптимизация бизнес-показателей салона красоты
📌 Описание проекта
В качестве управляющего салоном красоты я столкнулся с проблемой разрозненности данных и отсутствия прозрачной аналитики по возвращаемости клиентов и эффективности мастеров.
Цель проекта: Построить систему аналитической отчетности на основе SQL для выявления «точек роста» бизнеса, сегментации клиентской базы и оптимизации складских запасов.
🛠 Стек технологий
•	СУБД: PostgreSQL (использование сложных CTE, Window Functions).
•	Язык анализа: Python (Pandas, Matplotlib/Seaborn для визуализации).
•	Инструменты: Git, Jupyter Notebook.
📊 Какие задачи были решены:
1. Когортный анализ (Retention Rate)
Проблема: Было непонятно, на каком месяце клиенты перестают ходить в салон.
Решение: Написал SQL-скрипт с использованием CTE и Window Functions (DENSE_RANK), который группирует клиентов по месяцу первого визита и отслеживает их активность в динамике.
•	Результат: Выявлено падение удержания на 3-й месяц, что послужило поводом для внедрения CRM-рассылки с бонусами.
2. Рейтинг и эффективность мастеров
Проблема: Несправедливое распределение премий.
Решение: Реализовал расчет доли каждого мастера в выручке салона с помощью оконных функций (SUM() OVER). Ранжировал мастеров по KPI (средний чек, возвратность к конкретному мастеру).
•	Результат: Внедрена прозрачная система KPI, увеличившая выручку на 10% за счет мотивации персонала.
Благодаря переходу на data-driven подход, удалось не только систематизировать хаотичные данные, но и принять конкретные управленческие решения, которые привели к росту прибыли.
